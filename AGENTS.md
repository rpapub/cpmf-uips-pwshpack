# AGENTS.md

Guidance for coding agents (Claude Code, Copilot, Cursor, and similar) working
in this repository. Follow this file before making any changes.

---

## Orientation — run these first

Before reading any source file, run these commands to build a live picture of
the module. Prefer observed output over assumptions from file names.

```powershell
# What is exported?
Get-Command -Module CpmfUipsPack | Sort-Object Name

# What does each function do?
Get-Command -Module CpmfUipsPack | ForEach-Object { Get-Help $_.Name -Full }

# Dry-run the main entry point — traces the full execution path without side effects
Invoke-CpmfUipsPack -ProjectJson 'C:\repos\AnyProject\project.json' -WhatIf
```

`-WhatIf` is the fastest way to understand what `Invoke-CpmfUipsPack` actually does
end-to-end. Read its output before reading source files.

```

  Public/       ← the 7 exported functions — start here
  Private/      ← internal helpers; never call these from outside the module
  tests/        ← Pester 5 test suite; mirrors Public/ structure
```

## Verify before and after every change

```powershell
# Full test suite — must stay at 0 failures
Invoke-Pester ./CpmfUipsPack/tests/ -Output Normal

# Linter — must stay at 0 warnings
Invoke-ScriptAnalyzer -Path ./CpmfUipsPack -Recurse -Settings ./CpmfUipsPack/PSScriptAnalyzerSettings.psd1

# Manifest integrity
Test-ModuleManifest ./CpmfUipsPack/CpmfUipsPack.psd1
```

Run all three before touching any file. Run all three again after. If any of the
three was clean before your change and is not clean after, fix it before stopping.

## Rules agents must follow

**Do not break the public surface.**
The 7 exported functions are the contract. Renaming a parameter, changing a
default value, or removing a function is a breaking change. Avoid unless explicitly
instructed.

**New public functions must be added to `FunctionsToExport` in `CpmfUipsPack.psd1`.**
The manifest is not auto-discovered — omitting a function silently hides it.

**Private functions must not appear in `FunctionsToExport`.**
They are loaded by the module but must not be callable from outside it.

**`docs/reference/` and `CpmfUipsPack/en-US/` are generated — do not edit.**
Regenerate them with platyPS after any comment-based help changes:
```powershell
Import-Module ./CpmfUipsPack/CpmfUipsPack.psd1 -Force
New-MarkdownHelp -Module CpmfUipsPack -OutputFolder ./docs/reference -Force
New-ExternalHelp ./docs/reference -OutputPath ./CpmfUipsPack/en-US -Force
```

**All state-changing functions must have `[CmdletBinding(SupportsShouldProcess)]`.**
This enables `-WhatIf` and `-Confirm`. Do not skip it for convenience.

**User-facing messages go to `Write-Host` or `Write-Verbose`, never to the pipeline.**
Pipeline output is machine-readable (`[string[]]` paths, thrown exceptions for errors).
Mixing messages into pipeline output breaks callers and the future CLI wrapper.

**Config keys are layered — do not hardcode defaults inside logic.**
The config hierarchy (user config → env vars → project config → parameters) is
load-bearing. When adding a new configurable value, wire it through
`Get-CpmfUipsPackEffectiveConfig` and the env var map, not just as a parameter default.

## Config hierarchy (understand this before touching config-related code)

```
Priority (lowest → highest)
────────────────────────────────────────────────────────────────────────────────
 1. User config   %LOCALAPPDATA%\cpmf\CpmfUipsPack\config.psd1
 2. Env vars      UIPS_FEEDPATH, UIPS_TARGETS, UIPS_NO_BUMP, ...
 3. Project cfg   -ConfigFile .\uipath-pack.psd1
 4. Parameters    -FeedPath, -Targets, -NoBump, ...   ← always win
────────────────────────────────────────────────────────────────────────────────
```

Merging happens in `Private/Get-CpmfUipsPackEffectiveConfig.ps1`.
Applying to unbound parameters happens at the top of `Invoke-CpmfUipsPack`.
These two steps are intentionally separate — do not collapse them.

## Test patterns

All tests use **Pester 5** with `InModuleScope CpmfUipsPack`.

- Variables set in `BeforeEach` must be passed into `InModuleScope` via
  `-Parameters @{ key = $script:var }` and received with `param($key)`.
  Direct `$script:` access does not work inside `InModuleScope`.
- Mock `Invoke-UipcliPack` (not `&`) to control uipcli exit codes.
- Mock `Invoke-WithFileLock` as a pass-through:
  `Mock Invoke-WithFileLock { param($LockFile, $ScriptBlock); & $ScriptBlock }`
- Use a temp directory per test (generated in `BeforeEach`) so tests are isolated
  and leave no state on disk.

## What this module is designed to become

This module is the backend of a future CLI tool (not yet built). Architectural
decisions already made with that in mind:

- Public functions return `[string[]]` paths or throw — no interactive prompts
- All output is suppressible (`Write-Host`/`Write-Verbose`, never pipeline)
- Settings are injectable via `UIPS_*` env vars — no filesystem required
- `-WhatIf` works end-to-end across all state-changing operations

When the CLI wrapper arrives, it will drive this module via env vars and
`-ConfigFile`, not by building PowerShell parameter plumbing. Do not add
interactive behaviour that would break headless operation.

## When a user reports a problem — collect and post diagnostics

If the user describes an installation or runtime failure, ask them to run:

```powershell
Get-CpmfUipsPackDiagnostics
```

The output is already pseudonymized and safe to share publicly. Once you have it,
check whether `gh` is available and authenticated before deciding how to post:

```powershell
# Check availability
gh auth status
```

If `gh` is available and authenticated, offer to post the diagnostics as a comment
on the open help-wanted issue and ask the user to confirm before doing so:

```powershell
# Post to issue #1 — only after user confirms
gh issue comment 1 --repo rpapub/cpmf-uips-pwshpack --body "<paste diagnostics output here>"
```

If `gh` is not available or not authenticated, show the user the manual URL:
`https://github.com/rpapub/cpmf-uips-pwshpack/issues/1`

Always ask the user to confirm the content before posting — never post diagnostics
automatically without explicit approval.
